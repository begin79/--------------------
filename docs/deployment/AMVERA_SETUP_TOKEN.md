# Настройка токена бота в Amvera

## Проблема
```
ModuleNotFoundError: No module named 'app.config'
```

Это происходит потому, что файл `app/config.py` не попадает в репозиторий (он в `.gitignore` для безопасности).

## Решение: Использовать переменные окружения

### Шаг 1: Настройте переменную окружения в Amvera

1. Войдите в панель управления Amvera
2. Перейдите в ваш проект → **Настройки** → **Переменные окружения** (Environment Variables)
3. Добавьте новую переменную:
   - **Ключ**: `BOT_TOKEN`
   - **Значение**: ваш токен бота (например: `8379788668:AAFIisg9BX3hdCONTQGenezZ9io3OPixb8A`)
4. Сохраните изменения

### Шаг 2: Перезапустите приложение

После добавления переменной окружения:
1. Перейдите в раздел **Развертывание** (Deploy)
2. Нажмите **Перезапустить** (Restart) или дождитесь автоматического перезапуска

### Шаг 3: Проверьте логи

Убедитесь, что приложение запустилось без ошибок:
- Перейдите в **Логи** (Logs)
- Проверьте, что нет ошибок `ModuleNotFoundError`

## Альтернативное решение: Создать config.py на сервере

Если вы предпочитаете использовать файл `config.py`:

1. В панели Amvera перейдите в **Файлы** (Files) или используйте SSH
2. Создайте файл `/app/app/config.py` со следующим содержимым:

```python
import os

TOKEN = os.getenv("BOT_TOKEN", "ваш_токен_здесь")
BASE_URL_SCHEDULE = "https://kis.vgltu.ru/schedule"
BASE_URL_LIST = "https://kis.vgltu.ru/list"
BASE_URL_VGLTU = "https://vgltu.ru"
```

⚠️ **Внимание**: Этот файл будет перезаписан при следующем деплое, если он не в репозитории!

## Рекомендация

✅ **Используйте переменные окружения** (Шаг 1) - это безопаснее и удобнее:
- Токен не попадает в код
- Легко менять без пересборки
- Не теряется при обновлении кода

## Проверка

После настройки токена, бот должен запуститься. Проверьте логи:
- Не должно быть ошибок `ModuleNotFoundError`
- Не должно быть ошибок `BOT_TOKEN не задан`
- Бот должен подключиться к Telegram API
